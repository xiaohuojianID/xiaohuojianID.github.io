OpenClash 是 OpenWrt 路由器上最流行的代理客户端之一，基于 Clash 核心开发。本教程将带您从零开始在 OpenWrt 路由器上安装和配置 OpenClash，实现局域网全局代理。

OpenClash 安装配置教程

适用于 OpenWrt / LEDE / ImmortalWrt 等固件

什么是 OpenClash？

OpenClash 是一个运行在 OpenWrt 平台上的 Clash 客户端，它利用 Clash 核心为路由器提供代理功能。与在 PC 或手机上安装客户端不同，在路由器上部署 OpenClash 可以让 整个局域网 中的所有设备（电脑、手机、平板、智能电视、游戏机等）自动获得代理能力，无需每台设备单独配置。

OpenClash 的主要特点：

  • 全局代理 — 局域网所有设备自动生效，零配置
  • 规则分流 — 国内流量直连，国外流量代理，智能高效
  • 多核心支持 — 支持 Clash、Clash Meta、Clash Premium 等多个核心版本
  • 多种运行模式 — TUN 模式、Redir 模式、Fake-IP 模式满足不同场景
  • 可视化面板 — 通过 LuCI 界面管理，操作直观
  • 订阅支持 — 支持 Clash 订阅链接，自动更新节点

准备工作

前置条件

  • 一台已刷好 OpenWrt 固件的路由器（推荐 x86、R2S/R4S、树莓派等性能较好的设备）
  • 路由器需要有足够的存储空间（建议 256MB+），或已挂载外部存储
  • 确保路由器可以正常访问互联网
  • 建议先备份当前路由器配置

Step 1：安装 OpenClash

OpenClash 可以通过 LuCI 界面在线安装，或手动上传 IPK 安装包安装。

方法一：在线安装（推荐）

  1. 登录 OpenWrt 管理后台（默认 192.168.1.1）
  2. 进入 系统 → 软件包
  3. 点击 更新列表 刷新软件源
  4. 在过滤器输入 luci-app-openclash，点击查找
  5. 点击 安装 按钮等待安装完成
  6. 安装完成后，刷新页面左侧导航栏即可看到 服务 → OpenClash

方法二：手动安装 IPK

  1. 前往 OpenClash Release 页面 下载对应路由器架构的 IPK 包
  2. 通过 SCP 或 WinSCP 将 IPK 包上传到路由器的 /tmp 目录
  3. SSH 登录路由器，执行安装命令：opkg install /tmp/luci-app-openclash_*.ipk
  4. 安装完成后刷新 LuCI 页面即可

💡 提示

如果在线安装失败，通常是软件源配置问题。可以尝试更换软件源镜像，或使用手动安装方式。

Step 2：基本设置

安装完成后，进入 服务 → OpenClash 进行基本配置。

  1. 切换语言：在 OpenClash 设置页面右上角选择简体中文
  2. 核心选择：在 全局设置 → 版本更新 中下载 Clash Meta 核心（推荐）或 Clash Premium 核心
  3. 启用开关：在 全局设置 中勾选「启用 OpenClash」
  4. 端口设置：默认 HTTP 代理端口 7890，SOCKS5 端口 7891，一般无需修改
  5. 面板设置：开启 Dashboard（Yacd 或 Razord），方便在浏览器中切换节点

Step 3：订阅导入

将机场提供的 Clash 订阅链接导入 OpenClash：

  1. 进入 配置管理 → 订阅管理
  2. 在 订阅 URL 输入框中粘贴机场提供的 Clash 订阅链接
  3. 设置 更新间隔（建议 24 小时，单位：小时）
  4. 点击 保存配置
  5. 点击 更新订阅 按钮拉取节点配置
  6. 更新成功后，进入 运行状态 查看节点列表是否正常显示

如果机场只提供 SS/SSR/V2Ray 链接而没有 Clash 订阅链接，请先使用 订阅转换服务 转换为 Clash 格式。

Step 4：运行模式选择

OpenClash 提供三种运行模式，根据您的需求选择：

Redir 模式

使用 iptables 透明代理，只代理 TCP 流量，性能最优。适合不需要 UDP 代理的场景。

TUN 模式

基于 TUN 虚拟网卡，支持 TCP + UDP 全流量代理。适合游戏、视频通话等需要 UDP 的场景。

Fake-IP 模式

使用 Fake DNS，劫持 DNS 请求并返回虚拟 IP，减少 DNS 污染影响。兼容性较好，推荐新手使用。

模式选择建议：

  • 新手推荐：Fake-IP 混合模式（兼容性最好，多数场景表现良好）
  • 性能优先：Redir 模式（资源占用低，适合低配路由器）
  • 功能全面：TUN 模式（支持 UDP，适合游戏玩家）

在 全局设置 → 模式选择 中更改运行模式，保存并重启 OpenClash 生效。

Step 5：规则优化

OpenClash 默认内置了规则集，但您可以根据需求进行优化：

使用第三方规则集

在 配置管理 → 规则管理 中，可以替换默认规则为社区维护的优质规则：

  • ACL4SSR — 国内最流行的规则集，覆盖面广，更新及时
  • Loyalsoldier — 精简高效的规则集，适合低配设备
  • Hackl0us — GeoIP-based 规则，轻量快速

自定义规则

您也可以手动添加自定义规则，在 配置管理 → 配置文件管理 中编辑配置文件，在 rules: 段添加：

rules:
  # 自定义直连规则
  - DOMAIN-SUFFIX,example.com,DIRECT
  # 自定义代理规则
  - DOMAIN-SUFFIX,netflix.com,Proxy
  # IP 段规则
  - IP-CIDR,192.168.0.0/16,DIRECT

分流策略优化

在 策略组 设置中，可以配置：

  • 自动选择 — 根据延迟或丢包率自动切换最佳节点
  • 故障转移 — 当前节点不可用自动切换到备用节点
  • 负载均衡 — 将流量分配到多个节点，提高整体速度

⚠️ 注意事项

  • 修改配置后记得点击「保存配置」并重启 OpenClash 使更改生效
  • 规则过多会消耗路由器内存，建议选择适合设备性能的规则集
  • 不要在配置文件中混用不同格式的规则，保持统一
