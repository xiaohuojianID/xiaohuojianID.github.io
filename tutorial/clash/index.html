2

获取订阅链接

订阅链接是机场（代理服务商）提供给用户的配置文件地址，通常以 https:// 开头。登录你购买的机场网站，在用户中心或仪表盘页面中，找到"一键订阅"或"复制订阅链接"按钮。

🔒 安全提醒

订阅链接包含你的账户信息，请勿分享给他人。如不慎泄露，可在机场后台重置订阅链接。

3

导入订阅

下面以 Clash Verge 为例演示如何导入订阅：

  1. a 打开 Clash Verge，点击左侧菜单的 Profiles（配置）
  2. b 点击右上角的 Import（导入）按钮
  3. c 在弹出的输入框中粘贴你之前复制的 订阅链接
  4. d 点击 Save，Clash 将自动下载并解析订阅配置

💡 提示

在 Clash for Windows 中，操作路径类似：Profiles → 输入框粘贴链接 → Download。如果订阅需要转换，可以使用 subconverter 等在线转换工具。

4

选择节点

订阅导入后，在 Clash 主界面的 Proxies（代理）选项卡中可以看到所有可用节点。节点通常按地区或类型分组（如 香港、新加坡、日本、美国、台湾等）。

  • 自动选择（Auto/URL Test）：Clash 会自动测试各节点延迟并选择最快的节点
  • 手动选择：点击任意节点即可手动切换到该节点
  • 延迟测试：右键点击节点或分组，选择"延迟测试"查看各节点的实时响应时间

📌 一般建议选择延迟低、丢包率低的节点。观看流媒体时选择对应地区的节点可获得更好的解锁效果。例如看 Netflix 日区选日本节点，看 Disney+ 选美国节点或新加坡节点。

5

开启代理

节点选择完成后，需要开启系统代理才能正常使用。Clash Verge 的顶栏有一个开关按钮，点击即可开启代理。你也可以在 Settings 中配置代理端口（默认为 7890）和系统代理设置。

默认代理设置

  • HTTP 端口：7890
  • SOCKS5 端口：7891
  • 混合端口：7890
  • 允许局域网连接：可选

系统代理模式

  • Rule（规则）：根据规则分流，推荐
  • Global（全局）：所有流量走代理
  • Direct（直连）：所有流量直连

⚠️ 如果开启代理后无法上网，请尝试切换代理模式（Rule → Global → Direct），或者检查端口是否被占用。

6

TUN 模式开启（进阶）

TUN 模式 是 Clash 最强大的功能之一，它可以虚拟一张网卡，将所有系统流量（包括游戏、UWP 应用、命令行工具等）强制通过 Clash 代理，实现真正的全局代理。

开启步骤：

  1. 1. 进入 Settings → TUN Mode
  2. 2. 将 TUN Mode 开关切换为 开启 状态
  3. 3. 如果提示安装虚拟网卡驱动，点击 安装 并按照系统提示完成安装
  4. 4. 重启 Clash 使 TUN 模式生效

⚠️ 注意事项

TUN 模式会接管所有系统流量，开启后部分网络工具（如抓包工具、VPN 客户端）可能无法正常工作。如遇问题可暂时关闭 TUN 模式或切换到规则模式。

❓ Clash 常见问题

Clash Meta（现更名为 Clash.Meta）是原版 Clash 的一个高性能分支，增加了许多新特性：

  • 支持 TUN 模式（虚拟网卡，全局代理）
  • 支持规则集（rule-set），可引用外部规则文件
  • 支持 DNS 劫持和防 DNS 泄漏
  • 更好的内存管理和性能优化
  • 支持 mux、uTLS 等协议特性

目前大多数新项目（如 Clash Verge、Clash Nyanpasu）都已基于 Clash Meta 内核，建议新用户直接使用 Clash Meta 或基于其内核的 GUI 客户端。

可能的原因和解决方法：

  • 订阅链接已过期或失效：登录机场后台重新获取并导入
  • 网络问题导致下载失败：检查网络连接，尝试使用代理或切换网络环境后重新导入
  • 订阅需要转换：某些机场只提供 SS/SSR 格式，需要使用订阅转换工具转换为 Clash 格式
  • 订阅格式不兼容：确认使用的是 Clash 专用订阅链接（通常是 YAML 格式）
  • 软件版本过低：更新 Clash 到最新版本

更新订阅的方法：

  • 手动更新：在 Profiles 页面点击订阅右侧的刷新/更新按钮
  • 自动更新：在 Settings 中设置订阅自动更新间隔（推荐每 24 小时更新一次）
  • 重启时更新：部分客户端支持启动时自动更新订阅

建议开启自动更新功能，这样可以始终使用最新节点。如果你的机场节点经常变动，可以将更新间隔缩短到 12 小时。

Clash 的规则分流是其核心功能，可以通过编辑配置文件来实现精细化的流量管理：

  • 规则文件位置：Profiles → 右键编辑 / Config 文件夹中的 config.yaml
  • 规则语法：规则类型, 匹配条件, 策略（如 DOMAIN-SUFFIX, google.com, Proxy）
  • 常用规则类型：DOMAIN（域名）、DOMAIN-SUFFIX（域名后缀）、DOMAIN-KEYWORD（域名关键词）、GEOIP（地理位置）、IP-CIDR（IP 段）
  • 策略组：Proxy（代理）、Direct（直连）、Reject（拒绝）
  • 在线规则集：可使用 rule-provider 引用 GitHub 上的规则集，如 Loyalsoldier/clash-rules

新手建议直接使用机场提供的默认规则，或者使用经过优化的规则集（如 ACL4SSR），无需手动编写规则。

Clash 无法启动或闪退的常见解决方法：

  • 检查配置文件：配置文件中可能存在语法错误，使用 YAML 验证工具检查
  • 端口冲突：7890 等端口被其他程序占用，在 Settings 中修改端口号
  • 权限不足：尝试以管理员身份运行（Windows）
  • 杀毒软件拦截：将 Clash 添加到杀毒软件白名单
  • 内核版本不兼容：更新或降级 Clash 内核版本
  • 重装解决 99% 问题：卸载后删除残留配置文件夹，重新安装最新版本

如果以上方法都无法解决，可以查看 Clash 的日志文件（通常位于配置目录下的 logs 文件夹）获取详细的错误信息，或在 GitHub Issues 中搜索类似问题。
