订阅链接（Subscription Link）是机场为用户提供的节点配置入口。通过订阅链接，您可以一键导入所有节点，无需手动添加。本教程将详细介绍订阅链接的获取、导入、转换、合并与自动更新等全流程操作。

机场订阅链接使用教程

适用于 Clash、Shadowrocket、Sing-box、OpenClash 等主流客户端

什么是订阅链接？

订阅链接（Subscription URL）是一个以 https:// 开头的 URL 地址，由机场服务商为用户生成。当您在客户端中导入订阅链接后，客户端会自动从服务器拉取并解析节点配置，包括节点名称、地址、端口、加密方式、协议参数等所有信息。

订阅链接的核心优势：

  • 一键导入 — 不必手动逐个添加节点，粘贴链接即可完成配置
  • 自动更新 — 节点信息更新后，客户端可自动拉取最新配置
  • 统一管理 — 多设备使用同一订阅链接，配置统一
  • 格式标准 — 基于 Clash/Sing-box 等标准格式，兼容主流客户端

💡 提示

订阅链接通常绑定了您的账号和设备数量限制。请不要将个人订阅链接公开分享，以免被他人使用导致设备数超限或流量被盗用。

Step 1：获取订阅链接

  1. 登录您购买的机场官方网站
  2. 进入 用户中心 或 我的订阅 页面
  3. 找到 订阅链接 或 Clash 订阅 相关选项
  4. 点击 复制订阅链接 按钮（通常是一个以 https:// 开头的长链接）
  5. 部分机场提供多种格式的订阅链接（Clash、Sing-box、Surge 等），请根据您使用的客户端选择合适的格式

⚠️ 注意

如果机场只提供通用订阅链接（SS/SSR/V2Ray 格式），您需要先进行 订阅转换 才能在某些客户端中使用。

Step 2：在不同客户端导入

以下是在主流客户端中导入订阅链接的具体方法：

Clash Verge（Windows / macOS / Linux）

  1. 打开 Clash Verge 客户端
  2. 点击左侧 订阅 标签页
  3. 在 订阅 URL 输入框中粘贴订阅链接
  4. 设置 自动更新间隔（建议 24 小时）
  5. 点击 导入 按钮
  6. 点击 更新 拉取节点配置
  7. 更新成功后，在 代理 标签页选择节点

Shadowrocket（iOS）

  1. 打开 Shadowrocket 应用
  2. 点击右上角 + 添加配置
  3. 在 类型 中选择 Subscribe
  4. 在 URL 中粘贴订阅链接
  5. 可选：备注名称方便识别
  6. 点击 完成 保存
  7. 客户端会自动拉取并显示节点列表
  8. 在主界面选择所需节点，点击开关连接

Sing-box（全平台）

  1. 打开 Sing-box 客户端
  2. 进入 配置 或 Profiles 管理页面
  3. 点击 新建 或 添加配置
  4. 选择 远程配置（Remote）类型
  5. 粘贴订阅链接到 URL 字段
  6. 设置 自动更新间隔
  7. 保存后点击 更新 拉取配置
  8. 启用该配置即可使用

OpenClash（OpenWrt）

  1. 登录 OpenWrt 路由器管理后台
  2. 进入 服务 → OpenClash
  3. 导航到 配置管理 → 订阅管理
  4. 在 订阅 URL 粘贴链接
  5. 设置更新间隔（建议 24 小时）
  6. 点击 保存配置 后点击 更新订阅
  7. 更新完成后在 运行状态 查看节点

更详细的 OpenClash 配置请参考 OpenClash 安装配置教程。

Step 3：订阅转换

如果您的订阅链接格式不兼容当前客户端，或需要对配置进行自定义调整，可以使用订阅转换服务。

什么是订阅转换？

订阅转换是将一种格式的订阅链接转换为另一种格式的过程。例如，将通用 SS 链接转换为 Clash 格式，或将 Clash 格式转换为 Sing-box 格式。

常用订阅转换服务

subconverter

开源转换工具，支持 Clash、Sing-box、Surge、Shadowrocket、Quantumult X 等几乎所有客户端格式。支持自部署。

GitHub 仓库 →

sub-web

基于 subconverter 的 Web 前端，界面友好，操作简单。支持自定义规则、远程配置等高级功能。

GitHub 仓库 →

如何使用 subconverter 进行转换

  1. 访问公开的 subconverter 实例（如 https://sub.xeton.dev）或自建服务
  2. 在 订阅链接 输入框中粘贴原始订阅链接
  3. 选择 目标客户端（Clash、Sing-box、Surge 等）
  4. 可选：选择 远程配置（即规则集模板）
  5. 可选：开启 高级选项 进行更多自定义（如包括/排除节点、重命名、过滤等）
  6. 点击 生成订阅链接
  7. 复制生成的转换后链接，导入到客户端中使用

💡 提示

建议选择支持自部署的转换服务，或使用信誉良好的公开实例。避免将包含个人信息的订阅链接提交到不可信的服务。

Step 4：多订阅合并

如果您购买了多个机场的服务，可以将多个订阅链接合并为一个配置文件，方便统一管理和切换。

方法一：使用 subconverter 合并

  1. 打开 subconverter 转换页面
  2. 在订阅链接输入框中，每行输入一个订阅链接（支持多个链接）
  3. 选择目标客户端格式
  4. 生成转换后的合并链接
  5. 将合并链接导入客户端即可同时看到所有机场的节点

方法二：手动合并配置文件

  1. 分别更新每个订阅，获取各自的配置文件
  2. 使用文本编辑器打开配置文件
  3. 将多个配置文件的 proxies: 段合并到一起
  4. 在 proxy-groups: 中创建新的策略组引用所有机场节点
  5. 保存合并后的配置文件并使用

方法三：客户端多订阅支持

部分客户端原生支持多个订阅：

  • Clash Verge — 支持添加多个订阅，在代理页面可以按订阅分组查看节点
  • Shadowrocket — 支持添加多个订阅，节点会合并显示
  • Sing-box — 通过配置文件手动合并多个订阅源

Step 5：自动更新配置

为了确保节点信息始终最新，建议开启订阅自动更新功能。

各客户端自动更新设置

Clash Verge

订阅设置中开启「自动更新」，设置更新间隔（推荐 24 小时）。支持启动时更新和定时更新。

Shadowrocket

在订阅编辑页面开启「自动更新」，设置更新间隔。iOS 后台会定期刷新订阅。

Sing-box

配置中设置 update_interval 字段，单位为小时。支持后台自动更新。

OpenClash

在订阅管理设置更新间隔，支持 cron 定时更新。可以开启启动时自动更新订阅。

使用 cron 实现定时更新（OpenWrt 用户）

如果您使用 OpenWrt 且需要更灵活的更新策略，可以 SSH 登录路由器设置 cron 定时任务：

# 每天凌晨 3 点更新订阅
0 3 * * * /etc/init.d/openclash update

# 每天早晚各更新一次
0 6 * * * /etc/init.d/openclash update
0 18 * * * /etc/init.d/openclash update

编辑 cron：crontab -e，保存后重启 cron：/etc/init.d/cron restart

⚠️ 注意事项

  • 更新间隔不宜过短，建议至少 6 小时，避免频繁请求导致 IP 被限
  • 如果订阅链接变更，需要在客户端中手动更新链接
  • 自动更新依赖客户端的后台运行权限，确保客户端在后台不被杀死
  • 部分机场的订阅链接有有效期，过期后需要重新获取

常见问题 (FAQ)

可能的原因：1）订阅链接已过期，登录机场官网重新获取；2）链接格式与客户端不兼容，需要使用订阅转换服务转换格式；3）链接中包含特殊字符被截断，请完整复制；4）客户端版本过旧，不支持最新的配置格式。建议先在浏览器中打开订阅链接，看是否能正常下载配置文件。

是的，订阅链接过期后，客户端更新订阅时会返回错误或空配置，导致节点列表清空。但如果您关闭了自动更新，已缓存的节点配置可以继续使用，直到您手动更新或重启客户端。建议在套餐到期前及时续费并更新订阅链接。

可以但不推荐。大部分机场会限制同时在线设备数量（通常 2-5 台）。如果多台设备使用同一链接，可能会触发设备数超限导致无法连接。建议根据套餐允许的设备数合理分配，或在路由器（OpenClash）集中代理，局域网设备共享连接。

订阅转换本身不会影响节点速度。可能的原因：1）转换时选择了错误的协议或配置参数；2）转换服务添加了额外的规则或策略影响路由；3）部分机场对非官方订阅格式有限速；4）建议对比原始订阅和转换后订阅的速度差异，如果确实有影响，联系机场客服确认。

1）不要在公共场合（论坛、社交网络、群聊）分享订阅链接；2）使用支持鉴权的订阅转换服务，避免链接明文存储在转换记录中；3）定期更换订阅链接（大部分机场支持重置链接）；4）如果发现流量异常或设备数超限，立即重置订阅链接并修改密码；5）使用订阅转换时，选择自部署或可信的转换服务。

不同订阅格式对应不同的客户端：Clash 格式（YAML）适用于 Clash Verge、OpenClash、Clash Meta 等 Clash 生态客户端；Sing-box 格式（JSON）适用于 Sing-box 客户端；Surge 格式适用于 Surge（macOS/iOS）；Quantumult X 格式适用于 Quantumult X；通用格式（SS/SSR/V2Ray 链接）是一种基础格式，大多需要通过订阅转换才能在功能完整的客户端中使用。
